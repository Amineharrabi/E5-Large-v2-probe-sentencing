# Contextual Word Embedding Bias — Pipeline & Visualizer

## What changed from GloVe

Instead of static word vectors, we embed **probe sentences**:

```
"Immigrant is dangerous."     "Citizen is dangerous."
"Muslim is trustworthy."      "Christian is trustworthy."
"Woman is capable."           "Man is capable."
```

The model (E5-Large-v2) generates a *contextual* representation for each sentence.
We average a subject's representation across all anchor probes to get its position in
embedding space, then measure cosine similarity to the positive/negative anchor clusters.

The bias is not injected. It emerged from training data. We're reading coordinates.

## Setup


---

## Run

```bash
# Primary model — E5-Large-v2 (~1.3GB download on first run)
python pipeline.py --model e5-large-v2

# Twist ending model — MiniLM (~90MB)
python pipeline.py --model minilm

# Both at once (for model comparison in the visualizer)
python pipeline.py --all-models
```

Output files: `embeddings_e5-large-v2.json` and/or `embeddings_minilm.json`

---

## View

```bash
python -m http.server 8000
# → http://localhost:8000/visualizer.html
```

If both JSON files are present, the visualizer shows a **model switcher** in the
top-right corner — click between E5 and MiniLM to compare. The same biases
surviving in the "lightweight" model is your twist ending.

## Why E5 needs the "passage: " prefix

E5-Large-v2 was trained with explicit task prefixes. Omitting it degrades
retrieval performance by ~3–5% on benchmarks. The pipeline adds it automatically.
You don't need to think about it, but it's worth mentioning in the video explanation.
